可视化算法

理解复杂数据结构的最好方法是看到它们的实际作用。我们已经为各种数据结构和算法开发了交互式动画。 我们的可视化工具是使用 HTML5 canvas 元素用 javascript 编写的， 并且几乎可以在任何现代浏览器中运行 -- 包括 iPhone 和 iPad 等 iOS 设备，甚至是 Kindle 中的 web 浏览器！ （Kindle 中的帧速率足够低，因此可视化效果并不十分有用，但基于树的可视化效果 -- BSTs 和 AVL 树 -- 似乎足够好）


检查 算法 菜单所有最新的 javascript 实现的。

算法专用控件

屏幕顶部（上述屏幕截图中的红色框）是特定于算法的控件 -- 这些控件将根据你要可视化的算法而改变。 在此示例中，你可以在 BST 中插入，删除或查找元素，方法是在适当的字段中输入文本，然后按回车键或单击相关按钮。 单击打印按钮可以打印树。发出命令后，可视化将开始，并且可以由屏幕底部的常规动画控件进行控制。

常规动画控件

  • 跳过如果你在动画的中间，则此按钮将完全撤消当前命令。如果你不在动画中间，则此按钮将撤消上一个命令
  • 后退仅当你暂停了当前动画（使用播放/暂停按钮）时，此按钮才处于活动状态。在当前动画中退后一步。 如果你当前没有动画，请退回到上一个命令。你可以使用此按钮（有足够的按键）来备份所有已完成操作的全部历史记录
  • 播放/暂停在播放模式（算法可以免费运行直到完成）之间切换和暂停模式（你需要按"前进"或"后退"按钮以推进动画）
  • 前进仅当你暂停了当前动画（使用"播放/暂停"按钮）并且当前动画尚未完成时，此按钮才处于活动状态。在当前动画中前进一个步骤。
  • 向前跳过仅当当前动画尚未完成时，此按钮才处于活动状态。跳到当前动画的末尾
  • 动画速度（滑块）更改动画的速度。此滑块的值保存在 cookie 中，因此，如果你具有首选速度，则只需设置一次即可
  • w, h, Chnage 画布大小更改显示区域的宽度/高度。当更改立即生效时，动画将不会在此新字段中居中，直到你重新加载页面。 这将清除当前动画，但是 width 和 height 值也保存在 cookie 中，因此你只需要更改一次此字段， 然后如果你使用的是智能手机或台式机，并且屏幕负载很大，一切都应该正常工作。
  • 移动控件在网页顶部或底部的常规控件之间切换

    • 请注意，通过向后移动动画会很容易使自己感到困惑 -- 你可能会将动画的下一步与上一步混淆，并误解了算法的工作原理。 你可能希望只在第一次研究特定算法时才向前走。

Java/Swing 版本

这项工作基于我们使用 Swing 在 Java 中创建的可视化效果 -- 尽管不再维护该代码，但该代码仍在此处可用。 我们尚未将一些算法移植到新系统中，因此较旧的版本可能仍然有用。

Flash 版本

我们还开发了可视化的 flash 版本，其中包含用 javascript 开发的算法的子集。 flash 版本不如主要的 javascript 版本受支持，但是我们将为那些可能比 javascript 更好地使用 flash 框架的人提供 flash 版本。 但是，javascript 版本是首选版本，并且最快会得到更新和错误修复。
